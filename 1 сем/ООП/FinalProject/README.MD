Основные возможности:
    хранение данных в памяти приложения, с сохранением кошелька в файл при выходе
    поддержка нескольких пользователей (логин/регистрация)
    ведение категорий бюджета
    добавление доходов и расходов
    установка бюджета по категориям
    автоматические уведомления при превышении бюджета категории
    вывод отчётов в терминал и экспорт в файл
    подсчёт доходов, расходов, баланса, включая разрез по категориям
    валидация пользовательского ввода

Архитектура
    Wallet	          	объект доменной модели кошелька
    Transaction	      	операция дохода / расхода
    CategoryBudget	  	категория расходов с бюджетом
    WalletService	  	бизнес-логика управления кошельком
    AuthService	      	авторизация / хранение пользователей
    UserStore	      	хранилище пользователей (пароли в PBKDF2 hash)
    PersistenceService	сохранение/загрузка Wallet в JSON
    ReportService	  	формирование отчётов
    ConsoleUI	      	интерфейс взаимодействия через терминал
    Main	          	точка входа и цикл обработки команд

Все данные хранятся в оперативной памяти, а при завершении работы кошелёк текущего авторизованного пользователя записывается в JSON.

При первом запуске будет создана директория data (если отсутствует).

Пример команд
	register имя пароль				регистрация пользователя
	login имя						вход (пароль будет запрошен)
	addcat FOOD						добавить категорию FOOD
	setbudget FOOD 200				установить бюджет 200 на FOOD
	income 300 GENERAL зарплата		добавить доход
	expense 50 FOOD groceries		расход 50 по категории FOOD
	summary							полный отчёт
	saveout report.txt				экспорт отчёта в файл
	exit							выход с сохранением кошелька
	
Пример сценария
	register ivan qwerty123
	login ivan
	addcat FOOD
	setbudget FOOD 150
	income 500 GENERAL salary
	expense 120 FOOD groceries
	summary
	exit
