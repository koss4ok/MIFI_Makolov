Проект представляет собой консольное приложение на Java, которое позволяет пользователям создавать короткие ссылки на основе длинных URL и открывать их.
Каждый пользователь идентифицируется по UUID. Для каждой короткой ссылки задаётся:
1. Срок жизни (в днях)
2. Максимальное количество переходов
3. Автор

Все данные хранятся в локальном CSV файле (DB.csv).
Короткие URL генерируются по хешу SHA-256(userId + fullUrl) и берутся первые 10 символов.

Основная логика
При старте пользователь проходит авторизацию:
	можно ввести существующий UUID
	или просто нажать Enter → создаётся новый
	
Пользователь вводит ссылки в консоль:
	если введена длинная ссылка → создаётся короткая
	если введена короткая ссылка → выполняется попытка открытия оригинальной
	
При открытии ссылки выполняется проверка:
	не истёк ли срок действия
	не превышен ли лимит переходов
	(Если условие нарушено → запись удаляется из CSV.)

Дополнительно доступны команды:

	? — справка
	Switch — смена пользователя
	Exit — выход

Тестирование

В проекте написано 17 тестов:
9 тестов для UrlInfo, 8 тестов для Main.

Что тестируется:
	корректная запись в CSV,
	чтение CSV
	детерминированность хеша
	срок действия ссылки
	лимит ссылок
	фильтрация по автору
	работа фабрики createShortUrl()
	работа isShort(), auth(), Switch() (без запуска консольного приложения в цикл)
	корректное поведение при разных вариациях пользовательского ввода

CSV для тестов создаётся рядом с оригинальным, в файле DB_test.csv, чтобы не портить рабочий DB.csv.

Запуск тестов
	через IntelliJ (правый клик → Run Tests)
	или через Maven:
	mvn test
